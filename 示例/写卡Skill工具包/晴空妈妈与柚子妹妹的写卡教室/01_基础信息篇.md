# 第一章：基础信息篇

## 晴空妈妈与柚子妹妹的写卡教室

---

**🍊 柚子**：晴空妈妈！我想写一个角色！我有好多想法！她特别可怜，特别善良，特别自卑，然后她家里特别穷，妈妈生病了需要筹钱，然后她的同桌是个很烦人的富二代——

**☀️ 晴空**：等等，小柚子。深呼吸。

**🍊 柚子**：嗯？

**☀️ 晴空**：你刚才说了好多"特别"。特别可怜，特别善良，特别自卑。妈妈想问你一个问题。

**🍊 柚子**：什么问题？

**☀️ 晴空**：如果你把"特别可怜、特别善良、特别自卑"这几个词遮住，换成别的角色名字，这些形容词还能成立吗？

**🍊 柚子**：嗯……好像……很多角色都能用？

**☀️ 晴空**：对。这就是问题所在。你现在脑子里有的是"想法"，还不是"角色"。想法是模糊的、谁都能用的、飘在空中的。角色是具体的、只属于她的、落在纸面上的。

**🍊 柚子**：那我应该从哪里开始？

**☀️ 晴空**：从最朴素的地方开始。先给她一张身份证。

---

## 一、为什么要先写基础信息

**☀️ 晴空**：小柚子，你知道为什么我们要先写基础信息，而不是直接冲上去写性格吗？

**🍊 柚子**：因为……要让AI知道她是谁？

**☀️ 晴空**：说对了一半。另一半的原因更重要：让你自己知道她是谁。

很多新手作者一上来就急着写性格、写情绪、写她有多悲惨。可你还没想清楚她住在哪里、家里几口人、每天怎么去上学。这些"无聊"的小事才是角色站起来的地基。

基础信息回答的问题很简单：这个人是谁，长什么样，经历过什么，和{{user}}什么关系。

**🍊 柚子**：就这些吗？感觉好简单啊。

**☀️ 晴空**：看起来简单，90%的人在这一步就开始犯错了哦。妈妈一个一个教你。

---

## 二、基础信息的结构

**☀️ 晴空**：角色基础信息分四个部分。

```yaml
角色档案:
  基本信息:
  外貌特征:
  背景设定:
  关系设定:
```

记住一件很重要的事：**性格不在这里写。**

**🍊 柚子**：啊？可是我看很多人都把性格写在最前面诶……

**☀️ 晴空**：这就是他们角色标签化的原因之一。

你想想看，如果你在基础信息里就写了"极度自卑、沉默寡言、极致善良"，AI还没读到你后面精心写的调色盘和衍生，就已经对这个角色定型了。它会用最刻板的方式去演绎"极度自卑"这四个字。然后你后面再怎么写，都是在和前面的标签打架。

**🍊 柚子**：哦……那我把性格单独放在后面？

**☀️ 晴空**：对。基础信息只回答"这个人是谁"，不回答"这个人是什么样的"。这两个问题看起来像一回事，其实完全是两件事。

"她17岁，高二，住在老旧小区的顶楼加盖房里"——这是"她是谁"。

"她极度自卑、沉默寡言"——这是"她是什么样的"。

前者放这里，后者放调色盘里。分开放，各管各的。

---

## 三、基本信息：给她一张身份证

**☀️ 晴空**：这部分最简单，就是填表。

**🍊 柚子**：那我写！

```yaml
基本信息:
  姓名: 陈乖乖
  年龄: 17岁
  性别: 女
  身份: 高二学生
  与{{user}}关系: 同班同学，坐在{{user}}前面一排
```

**☀️ 晴空**：很好。简单、干净、有用。

注意"与{{user}}的关系"这一行，你只需要一句话的定位。具体的关系展开放在最后的"关系设定"里。

**🍊 柚子**：那如果我的角色有什么特殊身份呢？比如她是学霸？

**☀️ 晴空**：写。"年级成绩前列"可以放在身份里。但"她之所以学习好是因为要拿奖学金维持生活"——这种解释放在背景设定里，这里只放结果。

---

## 四、外貌特征：90%的人都写废的部分

**☀️ 晴空**：好了小柚子，现在写外貌。你先按你的想法写，然后妈妈帮你改。

**🍊 柚子**：好嘞！让我想想……

```
外貌特征:
  整体印象: 一个总是低着头、缩着肩膀的女孩子，像一株生长在墙角的小草。
  面容: 五官极其精致秀气，是古典画中走出来的美人胚子。
  眼睛: 非常纯粹的黑色瞳仁，又大又亮，像含着一汪清泉，天生带有无辜惹人怜爱的神情。
  皮肤: 病态的、缺乏血色的雪白，皮肤非常细腻。
  身材: 体重很轻，骨架纤细，单薄得像一折就断。
  头发: 黑色长发，因为营养问题发色偏黄，没有发型可言。
  穿着: 永远是那套洗得发白的校服。
```

**☀️ 晴空**：好，妈妈现在告诉你，这段里面有多少东西是废话。

**🍊 柚子**：啊？！我写了好多的！

**☀️ 晴空**：妈妈不是说你写得少，是说这里面有很多字在做无用功。我们一条一条看。

"一个总是低着头、缩着肩膀的女孩子，像一株生长在墙角的小草。"

这是性格描写和文学比喻。"低着头、缩着肩膀"描述的是她的行为习惯，属于性格范畴。"像一株小草"是比喻，AI读了这个比喻只会学会用同样华丽的方式去描写她，帮不了你记住角色长什么样。

**🍊 柚子**：那这行应该怎么改？

**☀️ 晴空**：删掉。外貌不写行为习惯，不写比喻，不写"感觉"。白描、零度、干干净净。

继续看。"五官极其精致秀气，是古典画中走出来的美人胚子。"

妈妈问你：把这句话的名字遮住，放在任何一个"好看的女角色"身上，成立吗？

**🍊 柚子**：……成立。

**☀️ 晴空**：那就等于什么都没写。外貌写的是"特征"。特征的意思是：**只有这个角色才有的东西。**

"精致""好看""美人"放在谁身上都行。这些词给不了AI任何具体画面。

**🍊 柚子**：可是我的角色确实很好看啊，我不写AI怎么知道？

**☀️ 晴空**：妈妈告诉你一个原则。这个原则叫**特征差异化**。

AI有自己的数据库。你只需要写偏离数据库默认认知的部分。

什么意思呢？17岁中国女高中生，AI的数据库里默认就是年轻的、皮肤还可以的、头发是黑色的。你不需要重复这些默认值。你只需要写**偏离默认的地方**。

**🍊 柚子**：那我怎么判断什么要写什么不要写？

**☀️ 晴空**：就一条标准。**如果你把外貌描写遮住名字，能靠这些特征认出是谁——就写得好。认不出来，放在别的角色身上也成立——那就是废话，删。**

现在我们来重新写陈乖乖的外貌。

```yaml
外貌特征:
  体型: 一米六六，偏瘦，肩膀窄
  发型: 黑色长发，低马尾，普通黑皮筋扎的，刘海自己剪的，有些参差
  瞳色: 黑色，杏眼，眼尾微垂
  穿着:
    校内: 蓝白校服，洗到有些发白，袖口和领口能看出缝补过的痕迹
    校外: 灰色旧T恤，洗软的运动裤
  特征:
    - 手指关节有冻疮愈合后的暗红色痕迹
    - 左手腕一根褪色的红线手绳
```

**🍊 柚子**：哇……好短啊。可是这些真的够吗？

**☀️ 晴空**：你数一下。六个特征点。

一米六六偏瘦肩膀窄——锁定体型。黑色长发低马尾黑皮筋刘海参差——锁定发型，而且"自己剪的，参差"这个细节暗示了她的生活状态，同时只属于她。杏眼眼尾微垂——偏离默认眼型，需要写。校服洗到发白有缝补痕迹——一句话把经济状况全带出来了。手指冻疮痕迹和红线手绳——两个只属于她的标志物。

遮住名字，你认得出这是谁吗？

**🍊 柚子**：认得出！那个"自己剪的参差刘海"和"缝补过的校服"一看就是她。

**☀️ 晴空**：对。六个特征就画完了一个人。没有写"精致""病态""雪白""像清泉"，但读的人已经能看见她了。

**🍊 柚子**：那我原来写的"病态的雪白皮肤"呢？不写了吗？

**☀️ 晴空**：你想想，17岁中国女生，AI数据库里默认皮肤什么样？

**🍊 柚子**：嗯……正常偏白？

**☀️ 晴空**：对。如果她的皮肤没有特别到足以成为辨识特征的程度，就不需要写。你写"病态的雪白"，AI读了会怎么做？它会在每一段描写里都提她的皮肤，"她苍白的脸""她没有血色的肤色"——你的角色会被自己的皮肤淹死。

而且"病态的"这个词是你的判断，不是特征。"雪白"对17岁中国女生来说偏离默认值，如果你确定要写可以简单写"肤色偏白"。但如果她除了偏白之外没有什么特殊的皮肤特征（比如疤痕、胎记），这行就可以省掉。

**🍊 柚子**：我懂了！外貌只写特征，不写形容。

**☀️ 晴空**：再补充一点。看看你原来写的"身体行为"那些内容——"总是低着头""贴着墙根走""缩成一团"。这些很好，但它们属于调色盘的衍生或者三面性的身体行为模式，不放在外貌里。外貌只写她**站在那里不动**的时候，你看到的硬件。

## 五、背景设定：只写让她变成现在这个样子的事

**☀️ 晴空**：好，小柚子，现在写背景。你又先来。

**🍊 柚子**：嗯！

```
背景设定:
  出身: 出生于南方小城的一个普通工薪家庭。童年时期有过短暂的安稳。
  关键事件1: 10岁时父亲因意外去世，家庭陷入贫困，这是她性格转变的起点。
  关键事件2: 母亲带着她背井离乡来到大城市打工，过度劳累导致身体垮掉。
  关键事件3: 16岁，高一那年，母亲被诊断出严重的肾脏疾病，需要长期透析。
  关键事件4: 高二上学期，合适的肾源找到了，但必须在一周内缴清九万八的手术费。
  经济状况: 赤贫。住在城市边缘老旧小区的顶楼加盖房，每一分钱都用在母亲治疗上。
  社交状况: 没有朋友，长期营养不良，利用课余时间打零工。
```

**☀️ 晴空**：这版比性格那边好很多了。因为你在写具体的事，而不是写形容词。

但妈妈要帮你调整一下。你这里有两个问题。

第一个问题：有些背景只是填充，删掉角色也不会变。

"出生于南方小城的一个普通工薪家庭。童年时期有过短暂的安稳。"——你把这行遮住，角色有变化吗？

**🍊 柚子**：好像……没什么变化？

**☀️ 晴空**：删。背景只写**改变了这个人的事**。和她现在的状态无关的童年琐事不写。

第二个问题：你把"这是她性格转变的起点"也写进去了。

**🍊 柚子**：对呀，这样AI就知道为什么她变成这样了。

**☀️ 晴空**：这是一种解释。解释属于"二次解释"条目，不属于背景。背景只写事实。你只需要把事件摆在那里，AI自己能看出"父亲去世→家庭贫困→搬家→母亲生病"这条因果链。你不需要替它总结"这是性格转变的起点"。

写多了解释，AI反而会偷懒——它不再自己理解角色，只是照搬你的结论。

**🍊 柚子**：好！那我改一下。

**☀️ 晴空**：妈妈给你示范一份。

```yaml
背景设定:
  家庭背景:
    父亲: 10岁时因意外去世
    母亲: 独自带她来大城市打工，高一时确诊慢性肾衰竭（尿毒症期），每周透析2到3次
    住处: 城市边缘老旧小区顶楼加盖房，面积小，冬冷夏热
  经济状况: 所有收入用于母亲治疗，长期营养不良
  学业: 市重点高中，成绩年级前列，依靠奖学金和学费减免维持
  当前处境: 高二上学期，医院通知找到匹配肾源，需在一周内缴清手术费九万八千余元，所有常规借款渠道已耗尽
```

**🍊 柚子**：好干净！每一条我都知道它在做什么。

**☀️ 晴空**：对。你看，"10岁时父亲因意外去世"，"母亲确诊慢性肾衰竭"，"成绩年级前列依靠奖学金维持"——三条就画出了这个人现在为什么是这样的。

"父亲去世"解释了家庭为什么塌。"母亲确诊"解释了经济为什么压到极限。"成绩好靠奖学金"解释了她为什么那么拼命读书。

没有一条是废的。

**🍊 柚子**：那"她小时候爸爸骑自行车带她"那个记忆呢？那个很感人诶。

**☀️ 晴空**：那是一个好细节。但它属于混色画面或者调色盘衍生里的内容——你可以在某个衍生里写"她记得小时候爸爸说她是最乖的宝贝，这个记忆现在想起来比以前更沉"。放在背景里反而显得像在堆煽情素材。

背景只放"骨头"，肉在后面长。

## 六、关系设定：具体的画面，不要抽象的形容

**☀️ 晴空**：最后一块。关系设定。你写和{{user}}的关系。

**🍊 柚子**：好的！

```
关系设定:
  与{{user}}的关系:
    定位: 同班同学，座位一前一后
    情感状态: 在故事开始前没有任何私人交集
    她对{{user}}的印象: 安静、干净、家境应该不错但不张扬
    互动方式: 收发作业时递本子，仅此而已
```

**☀️ 晴空**：很好。简洁、具体、有画面感。

"收发作业时递本子"——这一句话比"他们之间没有任何私人交集"好十倍。前者是一个画面，后者是一个总结。AI读到画面会生成画面，读到总结只会复述总结。

妈妈帮你补一个小东西。

```yaml
关系设定:
  与{{user}}的关系:
    定位: 同班同学，她坐{{user}}前一排
    认识方式: 没有正式认识过，只有收发作业时递本子的接触
    她的观察: 注意到{{user}}从不参与班里对她的议论，用的文具质感好但没有大logo
    互动方式: 递本子时会小心不让手指碰到对方，偶尔感觉到{{user}}的目光但不敢确认
```

**🍊 柚子**：啊！"注意到{{user}}从不参与议论"这个好有意思！

**☀️ 晴空**：对。你看，"她对{{user}}的印象"你写了"安静、干净"——这是两个形容词。换成"注意到他从不参与议论""用的文具质感好但没有大logo"，就变成了两个具体的观察行为。

AI读到形容词会输出形容词。AI读到行为会输出行为。

你希望你的角色被AI演成一个会观察细节的人，就给AI具体的观察细节。你希望她显得敏锐但沉默，就写她"注意到了"但"没有说出口"。

**🍊 柚子**：那和陆昂的关系呢？要不要写在这里？

**☀️ 晴空**：陆昂是NPC，可以简短地提到。但注意，同样的原则——写具体行为，不写形容词。

```yaml
  与陆昂的关系:
    定位: 同桌
    互动方式: 他会把零食放在她桌上，她会用书本把东西推到桌角；他当众替她出头时，她会把头埋得更低
```

两句话，两个画面。AI读完就知道这段关系的动态是什么。

**🍊 柚子**：比我原来写的"她对陆昂充满了恐惧和本能的排斥"好好多……

**☀️ 晴空**："充满恐惧和本能的排斥"是你的总结。总结不如画面。你让AI自己看见"她用书本把零食推到桌角"，AI就能自己感受到她的回避。你不需要替AI下结论。

## 七、完整示例：陈乖乖的角色基础信息

## 八、小柚子的课后笔记

**🍊 柚子**：等等让我整理一下今天学到的！

1. **基础信息只回答"这个人是谁"，不回答"这个人是什么样的"。性格放后面的调色盘里。**

2. **基本信息就是填表。简单直白，一句话定位关系。**

3. **外貌只写特征。特征的意思是"只属于这个角色的东西"。遮住名字认得出就对了，认不出就是废话。**

4. **外貌里不写：比喻、文学描写、意象、感觉、行为习惯、性格相关的动作。**

5. **背景只写改变了这个人的事。和角色当前状态无关的不写。不加解释，不替AI总结因果。**

6. **关系写具体画面。不写"她对他充满恐惧"，写"他把零食放她桌上，她用书本推到桌角"。**

**☀️ 晴空**：满分，小柚子。

## 附：常见错误对照表

| 错误写法 | 问题在哪 | 正确做法 |
|---------|---------|---------|
| 外貌："五官精致秀气，美人胚子" | 放在谁身上都行，无辨识度 | 写具体特征："杏眼，眼尾微垂" |
| 外貌："病态的雪白皮肤" | "病态的"是判断词，"雪白"是文学修辞 | 如果确实偏白就写"肤色偏白"，没有特殊就不写 |
| 外貌："像一株生长在墙角的小草" | 比喻，AI读了只会模仿华丽写法 | 删掉，外貌不写比喻 |
| 外貌："总是低着头缩着肩膀" | 行为习惯属于性格/调色盘范畴 | 移到后面的调色盘衍生里 |
| 背景："这是她性格转变的起点" | 作者替AI下结论，AI会偷懒 | 只写事件本身，让AI自己理解因果 |
| 关系："她对他充满恐惧和排斥" | 抽象总结，AI只会复述 | 写具体互动："他把东西放她桌上，她用书本推走" |
| 基础信息里写性格标签 | AI提前定型，后面的调色盘打不过前面的标签 | 性格放调色盘条目，基础信息只放事实 |
