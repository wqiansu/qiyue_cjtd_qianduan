# 多阶段调色盘 EJS 工作流

> 本工作流帮你把手写的多阶段调色盘（各阶段底色/主色调/衍生/二次解释）整合为一个EJS条目。
>
> **前提：** 你必须已经完成MVU变量设计（有好感度/关系状态等阶段判定变量）。
> **核心：** 工作流只帮你完成EJS代码，不替你写调色盘人设内容。

---

## 你需要准备什么

在开始之前，你需要已经手写好以下内容：

1. **阶段划分** — 用什么变量划分阶段，每个阶段的触发条件
2. **各阶段的调色盘头部** — 底色/主色调/点缀（可以随阶段变化）
3. **各阶段的专属衍生** — 只在该阶段出现的行为描写
4. **跨阶段通用衍生** — 始终存在、不随阶段变化的行为描写
5. **各阶段的专属二次解释** — 对该阶段专属衍生的作者注释
6. **跨阶段通用二次解释** — 对通用衍生的作者注释
7. **总结**

如果你还没写好这些内容，先用写卡预设的"📋 多阶段调色盘"条目完成引导。

---

## 工作流步骤

### 第一步：确认MVU变量路径

你需要知道以下信息：
- 好感度的变量路径（如 `stat_data.角色名.好感度`）
- 关系状态的变量路径（如 `stat_data.角色名.关系状态`）
- 其他阶段判定变量的路径

### 第二步：写变量读取头

在条目最顶部，用EJS读取判定变量：

```javascript
<%_
if (typeof gw === 'undefined') var gw = getvar('stat_data.角色名.好感度', { defaults: 0 });
if (typeof rel === 'undefined') var rel = getvar('stat_data.角色名.关系状态', { defaults: '陌生人' });
_%>
```

**规则：**
- 必须用 `typeof` 防重复声明
- 必须用 `var`（不能用 const/let）
- 路径必须带 `stat_data` 前缀
- 变量名建议简短（gw、rel等），避免和其他条目冲突

### 第三步：写调色盘头部（随阶段变化）

```javascript
性格调色盘：人的性格就像调色盘，由多种性格衍生组合而成才是活生生的人

<%_ if (条件1) { _%>
底色：[阶段1底色]
主色调：[阶段1主色调]
性格点缀：[阶段1点缀]
<%_ } else if (条件2) { _%>
底色：[阶段2底色]
主色调：[阶段2主色调]
性格点缀：[阶段2点缀]
<%_ } else { _%>
底色：[最终阶段底色]
主色调：[最终阶段主色调]
性格点缀：[最终阶段点缀]
<%_ } _%>
```

### 第四步：写各阶段专属衍生

每个阶段的专属衍生用 `if` 包裹：

```javascript
<%_ if (gw < 250) { _%>
好奇衍生一：...
好奇衍生二：...
<%_ } _%>

<%_ if (gw >= 250 && gw < 500) { _%>
在意衍生一：...
在意衍生二：...
<%_ } _%>

<%_ if (gw >= 500 && rel !== '恋人') { _%>
喜欢衍生一：...
喜欢衍生二：...
<%_ } _%>

<%_ if (rel === '恋人') { _%>
恋人衍生一：...
恋人衍生二：...
<%_ } _%>
```

**规则：**
- 条件边界不能重叠不能遗漏
- 数值用 `<` `>=` `&&`
- 文本用 `===` `!==`

### 第五步：写跨阶段通用衍生

直接写在所有 `if` 块外面，不用任何包裹：

```
莽衍生一：...
心软衍生一：...
不怕衍生一：...
```

### 第六步：写二次解释部分

先写标题，然后按同样的if逻辑分阶段：

```javascript
对角色的理解与思考:

<%_ if (gw < 250) { _%>
  关于好奇的本质: |
    ...
<%_ } _%>

<%_ if (gw >= 250 && gw < 500) { _%>
  关于在意: |
    ...
<%_ } _%>

  关于莽和心软的关系: |
    ...（通用二次解释，不用if包裹）

  总结: |
    这就是[角色名]的性格调色盘...
```

### 第七步：设置世界书配置

条目配置：
- **触发策略**：蓝灯常驻（strategy: constant）
- **位置**：角色定义后（position: after_character_definition, order: 99）
- **递归**：不可递归 + 防止进一步递归

## 常见阶段判定模板

### 纯好感度

```javascript
<%_ if (gw < 250) { _%>
  // 初识期
<%_ } else if (gw < 500) { _%>
  // 熟悉期
<%_ } else if (gw < 750) { _%>
  // 暧昧期
<%_ } else { _%>
  // 深入期
<%_ } _%>
```

### 好感度 + 关系状态

```javascript
<%_ if (rel !== '恋人') { _%>
  <%_ if (gw < 500) { _%>
    // 非恋人 + 低好感
  <%_ } else { _%>
    // 非恋人 + 高好感
  <%_ } _%>
<%_ } else { _%>
  // 恋人
<%_ } _%>
```

### 多变量组合

```javascript
<%_ if (gw >= 500 && trust >= 30 && rel !== '恋人') { _%>
  // 好感高 + 信任足 + 非恋人 = 暧昧期
<%_ } _%>
```

## 自查清单

完成后检查：
- [ ] 所有变量有 typeof 防重复声明 + var
- [ ] 所有变量路径有 stat_data 前缀
- [ ] 使用 `<%_ _%>` 格式
- [ ] 条件边界无重叠无遗漏
- [ ] 通用内容在所有 if 外面
- [ ] 每个阶段的二次解释 if 条件和对应衍生的 if 条件一致
- [ ] 条目设为蓝灯常驻，order 99，防递归
