# PR：世界书蓝绿灯触发（常驻 / 关键词命中）

> 给「🌍 世界引擎」的「后台推演世界书」加入**按条目激活类型注入**的能力：
> 🔵 常驻（constant）条目恒注入，🟢 关键词（selective）条目仅在近期对话命中其关键词时才注入，
> 跟随酒馆世界书自身的配置，且每条可在本扩展里单独覆写。
>
> 版本：`2.2.0` → `2.3.0`
> 目标分支：`cloud`（已含「酒馆缓存与存档 v2.2.0」，本 PR 叠加其上）
> 提交构成（3 条）：
> 1. `feat(worldbook)` 蓝绿灯触发引擎 + 设置 UI
> 2. `fix(worldbook)` 设置页 `settings` 未定义崩溃修复
> 3. `feat(worldbook)` 控制台命中明细 + 文档

---

## 1. 背景与动机

现状：[`world-engine-worldbook.js`](../world-engine-worldbook.js) 的 `buildPromptSection()` 把**所有已勾选且未停用**的条目**一律原样注入**推演 prompt。
它调用酒馆的 `getSortedEntries()` 仅用来**列出**条目，只取了 `disabled`，把 `constant` / `key` / `keysecondary` / `selectiveLogic` 等**原生激活配置全部丢弃**。

后果：

- 世界书里的 🔵 常驻条目和 🟢 关键词条目被同等对待——要么全注入、要么手动逐条取消。
- 关键词条目失去了「**只在相关时才出现**」的意义，大世界书会把无关设定也一股脑塞进推演，浪费上下文、稀释重点。

本 PR 让推演世界书**尊重每条的激活类型**，与酒馆世界书的「蓝灯/绿灯」语义对齐。

---

## 2. 设计原则：解耦

关键决策——**复刻酒馆的关键词匹配规则，但「触发什么」由本扩展自己判定**：

- **跟随酒馆配置**：激活类型（🔵 常驻 / 🟢 关键词）、关键词（主/次）、匹配选项（大小写、整词）都**读自酒馆条目本身**，不另设一套。
- **自己触发、不监听酒馆**：命中判定只扫描**本扩展喂给推演的上下文**（近期对话），**不**调用酒馆的 `checkWorldInfo`、**不**监听 `WORLD_INFO_ACTIVATED`。
  原因：用户会在本扩展里**自定义让推演读哪些条目**（既有的勾选 = 候选池）；若改为监听酒馆的扫描结果，就把「推演读什么」和「酒馆正文扫到什么」**耦合**死了，二者上下文与目的并不一致。
- **可独立覆写**：每条目可在本扩展里强制「常驻 / 关键词 / 关闭」，不改动酒馆世界书。

> 默认**关闭**（`worldbookTrigger:false`）：不开启时行为与之前**完全一致**（已选条目全部注入），对现有用户零改变。

## 3. 复刻酒馆的关键词匹配（`world-engine-worldbook.js`）

判定核心是 `activationOf(entry, scanText, mode)`，返回 `{ active, reason }`：`active` 决定本轮是否注入，`reason` 是人类可读的原因（供控制台明细，见 §6）。`isEntryActive()` 是它的布尔包装（供筛选与测试）。判定流程：

| 激活类型 | 行为 |
|----------|------|
| 🔵 常驻（`constant:true`） | 恒注入 |
| 🟢 关键词（有 `key`） | 主关键词任一命中 → 再按次要关键词逻辑判定；无主关键词的非常驻条目不触发（与酒馆一致） |
| 🔗 向量（`vectorized`） | 本扩展不做向量召回，按「非关键词」处理（可手动覆写为常驻/关键词） |

次要关键词逻辑（`selectiveLogic`，数值与酒馆一致）：

| 值 | 含义 | 命中条件（主命中后） |
|----|------|------|
| `0` AND_ANY | 与·任一 | 次要关键词**任一**命中 |
| `1` NOT_ALL | 非·全部 | 次要关键词**未全部**命中 |
| `2` NOT_ANY | 非·任一 | 次要关键词**一个都不**命中 |
| `3` AND_ALL | 与·全部 | 次要关键词**全部**命中 |

单关键词匹配 `matchKey()`：

- 形如 `/pattern/flags` 的关键词按**正则**处理。
- 其余按 `caseSensitive` 决定大小写；`matchWholeWords` 的**整词匹配仅对 ASCII 单词生效**。
- **中文等无词边界**（`\b` 不可用）一律退回**子串匹配**——这正是本扩展（武侠中文场景）的合理默认，避免整词规则把中文关键词全部漏掉。

> 扫描文本来自 [`world-engine-evolution.js`](../world-engine-evolution.js)：`dialogueText || (userMsg + '\n' + aiMsg)`，即本轮真正喂给推演的近期对话。

## 5. 数据结构

`world_engine_worldbook_selection_<chatId>`（IndexedDB / localStorage，仍按聊天隔离）：

```jsonc
{
  "ids": ["书A::3", "书A::7"],   // 候选池：本插件可读取的条目（既有语义不变）
  "t": 1700000000000,            // 保存时间戳（FIFO 兜底用）
  "overrides": {                 // 每条触发覆写；缺省=auto(跟随酒馆)，不落盘
    "书A::7": "const"            // 'const'(强制常驻) | 'key'(强制关键词) | 'off'(关闭)
  }
}
```

向后兼容：老格式（纯数组、或无 `overrides` 的 `{ids,t}`）照常解析；取消勾选的条目其覆写在保存时自动清理，不堆积。

## 6. 用户怎么用

设置页 **「后台推演世界书」** 分区顶部新增开关 **「启用蓝绿灯触发（跟随酒馆世界书）」**：

- **关闭（默认）**：已选条目全部注入推演——与之前一致。
- **开启**：
  - 每条显示徽标：🔵 常驻 / 🟢 关键词（附关键词预览）/ 🔗 向量 / ⚪ 无关键词。
  - 🔵 恒注入；🟢 仅当近期对话命中其关键词才注入。
  - 每条右侧下拉可覆写：**跟随酒馆 / 强制常驻 / 强制关键词 / 关闭**。
- 勾选仍是「候选池」：未勾选的条目永不参与（无论灯色）。改完点「保存世界书选择」一并落盘。

**怎么确认命中（可见性）**：

- **控制台明细**：每轮推演时 F12 Console 打印一条折叠日志 `[世界引擎] 世界书蓝绿灯：X/Y 注入`，展开可见每条 `✓ 注入 / · 跳过 | 原因 | 书名 / 标题`，原因含 🔵常驻 / 🟢命中「关键词」/ 🟢未命中 / 覆写。
- **提示词查看**：面板「调试」页显示发送给 API 的完整 prompt（含 `========== 已选世界书条目 ==========` 段）与「导出 Prompt」按钮，可核对实际注入内容。

## 7. 边界情况

| 情况 | 处理 |
|------|------|
| 未开启触发 | `buildPromptSection` 忽略扫描文本，维持「全部已选注入」 |
| 扫描文本为空 | 🔵 常驻仍注入；🟢 关键词条目不命中（无文本可扫） |
| 关键词含中文 | 子串匹配（不套用 ASCII 整词规则），可正常命中 |
| `/regex/` 非法 | 该关键词判为不命中，不抛错 |
| 🔗 向量条目 | 默认不触发；可手动覆写 |
| 取消勾选后又改覆写 | 保存时只保留仍在候选池内的覆写 |
| 旧版选择记录（纯数组） | 照常解析，`overrides` 视为空 |

## 8. 测试

**离线逻辑测试**：`vm` 沙箱加载**真实**的 `world-engine-worldbook.js`，mock `WORLD_ENGINE_STORE` / `WORLD_ENGINE_API`，直接验证 `matchKey` / `isEntryActive` / 存储兼容，**28/28 全过**：

1. 常驻恒注入；关键词命中/未命中。
2. `selectiveLogic` 四个分支（AND_ANY / NOT_ALL / NOT_ANY / AND_ALL）。
3. `/regex/` 关键词；大小写敏感；中文子串（整词规则不误伤中文）。
4. 覆写优先级（const/key/off 覆盖原生灯色）。
5. 总开关关闭时 = 全部已选注入（与旧行为一致）。
6. 存储向后兼容（纯数组、缺 overrides）。

**在线验证**：在真实酒馆页面（已装本扩展）调用部署中的 `isEntryActive` / `matchKey`，对上述用例逐项核对一致；并在实际聊天里开启总开关、对话命中关键词后确认 🟢 条目进入推演、🔵 始终在。

另：改动的 JS 文件均通过 `node --check`。

> 测试脚本为离线验证用途，未纳入仓库（项目本身无测试框架）。

**建议人工回归（需在真实酒馆里跑）**：
- 关一本含常驻+关键词的世界书：开总开关 → 对话提到某关键词 → 确认对应 🟢 条目本轮进入推演、未提到时不进；🔵 始终在。
- 单条覆写为「关闭/强制常驻」生效。
- 群聊；多本世界书合并；刷新面板后勾选与覆写不丢。
