给 AI 看的报告

项目：src/此间天地前端 状态栏
使用场景：将 dist/此间天地前端/index.html 内容复制到酒馆正则文件的“替换为”中，只显示状态栏 UI，并从最新楼层变量读取数据。

一、正确的变量读取方式

状态栏应该调用酒馆助手变量 API 读取最新楼层变量：

getVariables({ type: 'message', message_id: 'latest' })

然后读取其中的：

stat_data

也就是：

const activeMessageOption = { type: 'message', message_id: 'latest' };
const data = _.get(getVariables(activeMessageOption), 'stat_data');

这点已经确认是正确方向。状态栏不显示变量时，不要第一时间怀疑是“读取变量 API 错了”，应该先确认打包后的内联 JS 有没有在酒馆正则/Markdown/HTML 处理管线里被改写坏。

二、这次真正的问题

状态栏不显示变量的根因不是读变量 API 错了，而是打包后的内联 JS 在复制进酒馆正则“替换为”后，被酒馆/Markdown/HTML 处理管线改写，导致脚本语法错误，根本没执行到变量读取阶段。

三、已经遇到过的两个具体问题

问题 1：压缩 JS 里的表达式被实体解码破坏

打包压缩后出现类似：

a&&gt(a,'status')

进入酒馆后可能被解码成类似：

a&>(a,'status')

导致：

Uncaught SyntaxError: Unexpected token '>'

修复方式：

在 webpack.config.ts 中对 此间天地前端 禁用 JS 最小化，避免生成容易被酒馆误处理的压缩表达式。

参考写法：

const avoid_html_entity_minify = /[\\/]此间天地前端[\\/]/.test(entry.script);

optimization: {
  minimize: !avoid_html_entity_minify,
}

问题 2：JS 字符串里的 HTML 实体被提前解码

源码里原本有：

const ESC_MAP = {
  '&': '&amp;',
  '<': '&lt;',
  '>': '&gt;',
  '"': '&quot;',
  "'": '&#39;',
};

复制进酒馆后，&#39; 被解码成 '，实际脚本变成非法 JS：

const ESC_MAP = { ..., "'": ''' };

导致：

Uncaught SyntaxError: Invalid or unexpected token

修复方式：

不要在会被复制进酒馆正则的内联脚本里直接写完整 HTML 实体字符串。改成运行时拼接：

const HTML_ENTITY_PREFIX = String.fromCharCode(38);
const ESC_MAP: Record<string,string> = {
  '&': HTML_ENTITY_PREFIX+'amp;',
  '<': HTML_ENTITY_PREFIX+'lt;',
  '>': HTML_ENTITY_PREFIX+'gt;',
  '"': HTML_ENTITY_PREFIX+'quot;',
  "'": HTML_ENTITY_PREFIX+'#39;',
};

四、后续排查 checklist

在酒馆主页面控制台检查状态栏 iframe：

const iframe = [...document.querySelectorAll('iframe')]
  .find(f => f.contentDocument?.querySelector('.th-status-wrapper'));

const script = [...iframe.contentDocument.scripts]
  .map(s => s.textContent || '')
  .sort((a,b) => b.length - a.length)[0];

new Function(script);

上面这一句应该不报错。如果报错，说明脚本还没执行到读取变量阶段。

继续检查：

typeof iframe.contentWindow.__deleteAvatar__;

应该是：

"function"

再检查最新楼层变量：

TavernHelper.getVariables({
  type: 'message',
  message_id: 'latest'
}).stat_data;

应该存在。

五、打包后本地建议校验

运行：

pnpm build

然后检查 dist/此间天地前端/index.html 内主脚本是否可解析。

同时不要出现这些危险片段：

&&gt
&>(
&#39;
&amp;
&lt;
&gt;
&quot;

尤其是完整 HTML 实体不要直接出现在 JS 字符串里。

六、大白话解释

这次不是状态栏不会读变量。

它其实已经按正确方法去读了：读最新楼层的 stat_data。

真正的问题是：把打包好的 HTML 整坨塞进酒馆正则“替换为”以后，酒馆会再处理一遍里面的内容。它把 JS 代码里某些看起来像 HTML 转义的东西偷偷还原了，结果 JS 被改坏了。

所以状态栏只显示了壳子，里面的脚本在半路直接语法报错，根本没跑到“读取变量并渲染”的地方。

现在修掉的核心就是：别让打包后的 JS 里出现会被酒馆误解码的危险写法。

包括压缩后的怪表达式，以及 &amp;、&#39; 这种 HTML 实体字符串。
